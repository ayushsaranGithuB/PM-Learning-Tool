# Writing Guide

## Voice

- conversational
- practical
- clear
- lightly opinionated
- never academic for the sake of sounding smart

Write like a builder explaining something useful to another builder.

## Lesson Rules

- Teach the idea, not just the terminology.
- Explain the problem before the framework.
- Prefer concrete examples over abstract definitions.
- Keep lessons original even when inspired by external sources.
- Use references to support the lesson, not dominate it.

## Good Patterns

- Start with a realistic scenario.
- Name the mistake beginners often make.
- Introduce one mental model worth remembering.
- End with reflection questions that force judgment.

## Avoid

- copying source language
- dense jargon
- vague motivational filler
- checklist content with no narrative

## Quality Checklist

Before publishing a lesson, ask:

- Would an engineer with no PM experience understand this?
- Did I explain why this matters?
- Did I include a real example?
- Did I make one key idea memorable?
- Did I add recommended reading?
- Did I add flashcards?
